Cheshire II Commands

delete_recs - Remove records from a CheshireII database

SYNOPSIS

delete_recs [-L logfilename] configfilename filename record_id1

DESCRIPTION

Delete_rec reverses the indexing work done with index_cheshire using the description included in the configuration file configfilename to un-index and remove from access one or more SGML records. The filename specified can be the <FILETAG> or <FILENAME> from the configuration file.

For each of the record_id's (internal record sequence numbers) the program deletes all of the index and component entries for the record. Cluster information is NOT removed in the current version. The SGML/XML record is "deleted" by marking it in the associator file, no actual physical deletion of the record is performed. Note that "deleted" records may be re-indexed if they are not physically removed.

LOG and ERROR INFORMATION

If the "-L" option is not used, then running the delete_recs program creates a file in the current directory called INDEX_LOGFILE. If the "-L" option is used then the logfile is given the name provided on the command line. If this file already exists when the program is run, new information is appended to the existing file. INDEX_LOGFILE will contain any errors or problems encountered in indexing.
