This Annex provides a set of guidelines for using attributes in the GEO Attribute Set, and specifies the mandatory combinations of attributes required by GEO clients and servers. In addition, the valid combinations of Use, Relation, and Structure Attributes are enumerated.

B.1. Attribute Combination Guidelines

GEO clients and servers are expected to conform to the following guidelines for attribute combinations:

  • GEO clients and servers must support the mandatory attribute combinations from the GEO Attribute Set. Additional attribute combinations may also be supported; valid combinations of Use and Structure and Structure and Relation Attributes are specified in this Annex.
  • GEO clients are required to supply a single Use, Relation, and Structure Attribute with each query term. Where appropriate, an optional Truncation Attribute may also be supplied.
  • If a GEO Server receives a query term with no supplied attributes, the default behavior is to search the text of all records for the words in the query term. This is equivalent to specifying a Use Attribute of Anywhere (1035), a Relation Attribute of Equal (3), and Structure Attribute of Word List (6).
